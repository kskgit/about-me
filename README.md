## 技術・学習について

### Q. 普段どんな風にキャッチアップしてますか？
**【要点】**
- 定期的に情報源を決めて追う
- 実際に手を動かすことを重視

**【詳細】**
- 情報収集：
  - 公式ドキュメント / GitHub Releases
  - 技術ブログ、X（旧Twitter）
- 学習方法：
  - 小さく試すサンプル実装
  - 個人開発への即時反映
- 目的：
  - 「知る」より「使える状態」にすることを重視しています

## スキル・特性について

### Q. 得意なことは何ですか？
**【要点】**
- 課題を整理してシンプルに設計すること

**【詳細】**
- 要件を言語化し、責務を分離する設計
- 「今必要な複雑さ」に抑える判断
- 個人開発や小規模チームでの実装・改善サイクル

---

### Q. 苦手なことは何ですか？
**【要点】**
- 不確実性が高い状態での即断

**【詳細】**
- 情報が揃っていない状態では一度整理の時間を取りたくなる
- 対策として：
  - 仮説ベースで進める
  - 早めにレビューや壁打ちを行う
- 「慎重さ」がデメリットにならないよう意識しています

## コミュニケーションについて

### Q. どんなコミュニケーションスタイルを好みますか？
**【要点】**
- 非同期＋ドキュメントベースを好む

**【詳細】**
- 好きなスタイル：
  - Slack / Issue / PR コメント
  - 事前に論点が整理されているミーティング
- 理由：
  - 考える時間を確保できる
  - 認識齟齬を減らせる
- ただし：
  - 緊急時や詰まりどころでは即時会話も重視します
