### Q. 得意だと思うことは何ですか？
**チームが破綻なくコードを書けるようなアーキテクチャ考案や開発環境整備が比較的得意です**

**【補足】**
- 部分最適化よりも全体最適化を優先します
- 将来的なニーズやチームの手間のバランスを考慮します
- 1回きりではなく、長期的な運用に耐えうる「手入れのしやすいコード」を書く
- 常に構造や持続可能性に目が向くことが多い気がします

- チームメンバーと雑談出来る程度の関係性

### Q. どんなコミュニケーションスタイルを好みますか
**【要点】**
- アサーティブなコミュニケーションを好む傾向があります

**【詳細】**
- 好きなスタイル：
  - 相手のバックグラウンドなどが分かると不確実性が下がり話しやすいと感じます
  - チャットだと温度感が伝わりづらいため会話ベースが好みです
- 理論より感情に重きを置く傾向があります
  - 例えば多少理論がずれていたとしても、それを頑張って正すより聞き流して相手のテンションを下げない事を優先します
    - もちろんずれ具合によりますが、その方がトータルで合理的だと考えています
